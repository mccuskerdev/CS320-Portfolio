CS 320 Reflection

## How can I ensure that my code, program, or software is functional and secure?
I can ensure my code is functional by testing it regularly as I build it instead of waiting until the end. In this course, unit testing helped me verify that each part of the program worked as expected. Writing tests for different cases also helped me catch errors early. For security, I think it is important to follow good coding practices, validate inputs, and make sure the program handles unexpected values without breaking.

## How do I interpret user needs and incorporate them into a program?
I interpret user needs by carefully reading the requirements and breaking them down into smaller parts. From there, I think about what the user actually expects the program to do and design the code around that. In this course, the project requirements helped guide how I built the Contact and ContactService classes so they matched the expected behavior.

## How do I approach designing software?
I approach designing software by starting simple and focusing on the main requirements first. I try to plan out the structure before writing code so everything is organized. Then I build and test each part step by step. This makes it easier to find issues and adjust the design as needed without getting overwhelmed.
